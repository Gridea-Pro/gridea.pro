完成[基础配置](./configuration/)后，现在来创建你的第一篇文章。

## 创建文章

1. 点击左侧边栏的 **文章** 进入文章管理
2. 点击右上角的 **新建文章** 按钮
3. 在编辑器中输入标题和正文内容

## 编辑器功能

Gridea Pro 内置了功能完善的 Markdown 编辑器：

- **实时预览** -- 左侧编辑，右侧即时渲染
- **图片管理** -- 拖拽上传或从剪贴板粘贴图片
- **快捷键** -- 支持常用 Markdown 快捷键（加粗、斜体、链接等）
- **代码高亮** -- 支持主流编程语言语法高亮

### 常用 Markdown 语法

```markdown
# 一级标题
## 二级标题

**加粗文本** 和 *斜体文本*

- 无序列表项
- 另一项

1. 有序列表
2. 第二项

[链接文字](https://example.com)

![图片描述](图片路径)

`行内代码` 和代码块：

​```javascript
console.log('Hello, Gridea Pro!');
​```
```

## 文章设置

在编辑器右侧的设置面板中，你可以配置文章的元信息：

| 设置项 | 说明 | 对应模板变量 |
|--------|------|-------------|
| 标题 | 文章标题 | `post.title` |
| 发布日期 | 发布时间 | `post.date` |
| 特色图片 | 文章封面图 | `post.feature` |
| 标签 | 文章分类标签 | `post.tags` |
| 置顶 | 是否在列表中置顶 | `post.isTop` |
| 列表中隐藏 | 是否从列表页隐藏 | `post.hideInList` |

:::tip
`post.date` 是格式化后的字符串，不是时间对象。在主题模板中直接输出 `{{ post.date }}` 即可，不要对其使用日期过滤器。
:::

## 预览文章

点击编辑器底部的 **预览** 按钮，Gridea Pro 会使用当前主题实时渲染文章，在内置浏览器中展示效果。

预览时 Gridea Pro 会执行完整的渲染流程：

1. 将 Markdown 转换为 HTML（即 `post.content`）
2. 使用当前主题的 `post.html` 模板渲染文章页
3. 生成临时静态文件并在内置浏览器中打开

:::note
预览使用的是完整渲染管线，所以预览效果与最终部署效果一致。如果预览中样式异常，通常是主题模板问题而非编辑器问题。
:::

## 保存与管理

- 文章会自动保存草稿
- 在文章列表中可以搜索、排序和批量管理文章
- 支持将文章设为草稿状态，草稿不会出现在生成的静态站点中

完成第一篇文章后，前往[发布部署](./deployment/)将博客发布到线上。
