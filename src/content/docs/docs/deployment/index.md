Gridea Pro 生成的是纯静态 HTML 文件，可以部署到任何支持静态文件托管的平台。本章介绍各种部署方式的配置方法。

## 部署方式对比

| 平台 | 费用 | OAuth 一键授权 | 自定义域名 | HTTPS | 全球 CDN | 配置复杂度 |
|------|------|---------------|-----------|-------|----------|-----------|
| [GitHub Pages](./github-pages/) | 免费 | 支持 | 支持 | 自动 | 有限 | 低 |
| [Vercel](./vercel/) | 免费套餐 | 支持 | 支持 | 自动 | 全球 | 低 |
| [Netlify](./netlify/) | 免费套餐 | 支持 | 支持 | 自动 | 全球 | 低 |
| [Gitee Pages](./gitee/) | 免费 | 支持 | 支持 | 手动 | 国内 | 低 |
| [Coding Pages](./coding/) | 免费 | 不支持 | 支持 | 自动 | 国内 | 中 |
| [自定义服务器](./custom-server/) | 取决于服务器 | 不适用 | 完全掌控 | 需配置 | 需配置 | 中-高 |

:::tip
GitHub / Gitee / Netlify / Vercel 四个平台都支持 **OAuth 一键授权**，免去了手动创建 Token、复制粘贴的环节。推荐首次配置时直接走 OAuth，详见 [OAuth 授权总览](./oauth/)。
:::

:::tip
对于大多数个人博客，GitHub Pages 是最简单的起步方案。如果需要更快的全球访问速度，推荐 Vercel 或 Netlify。
:::

## 通用部署流程

无论选择哪个平台，Gridea Pro 的部署流程都遵循相同的核心步骤：

1. **配置远程仓库** -- 在 Gridea Pro 设置中填写仓库地址和认证信息
2. **编写内容** -- 创建和编辑文章
3. **点击同步** -- Gridea Pro 自动渲染 + 推送
4. **平台构建** -- 托管平台检测到推送后自动部署（或直接提供静态文件服务）

## 部署前检查清单

在首次部署前，确认以下配置：

- [ ] 站点域名已正确设置（`config.domain`）
- [ ] 远程仓库 Token 有效且权限充足
- [ ] 主题在预览中正常工作
- [ ] 所有文章已保存（非草稿状态的文章才会发布）

:::note
Gridea Pro 在同步时会将整个静态站点推送到指定仓库的指定分支。请确保该分支专门用于博客部署，不要与源码仓库混用。
:::

## 自定义域名通用步骤

大部分平台的自定义域名配置流程相似：

1. 在域名 DNS 管理中添加 CNAME 记录，指向平台提供的域名
2. 在 Gridea Pro 设置中将站点域名改为你的自定义域名
3. 在托管平台的项目设置中添加自定义域名
4. 等待 DNS 生效和 HTTPS 证书签发（通常几分钟到几小时）

## 输出目录结构

了解 Gridea Pro 的输出结构有助于排查部署问题：

```
output/
├── index.html                # 首页
├── page/2/index.html         # 分页页面
├── post/<slug>/index.html    # 各文章详情页
├── archives/index.html       # 归档页
├── tags/index.html           # 标签汇总
├── tag/<name>/index.html     # 各标签页
├── styles/                   # CSS（从 assets/styles/ 复制，去除 assets/ 前缀）
├── media/                    # 媒体文件
└── 404.html                  # 错误页面
```
