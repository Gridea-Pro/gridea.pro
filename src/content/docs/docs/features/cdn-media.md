---
title: CDN 与图片管理
description: 配置图片 CDN 加速，支持又拍云、Cloudinary、GitHub 等多种存储服务。
---

当博客图片较多时，使用 CDN 可以显著提升页面加载速度。Gridea Pro 支持将图片自动上传到 CDN 服务，并在渲染时替换为 CDN 链接。

## 支持的 CDN 服务

| 服务 | 类型 | 特点 |
|------|------|------|
| **又拍云** | 国内 CDN | 国内访问速度快，需备案 |
| **Cloudinary** | 国际云存储 | 免费额度充足，自带图片处理 |
| **GitHub** | 代码托管 | 利用 GitHub 仓库存储图片 |
| **自定义 S3** | 兼容存储 | 支持 S3 协议的任意存储服务 |

## 配置 CDN

1. 在 **设置** 中找到 CDN 配置区域
2. 开启 **启用 CDN 上传** 开关
3. 选择 CDN 服务商
4. 填写服务商配置参数（各服务参数不同，见下方详解）
5. 设置自定义域名（可选）
6. 点击 **测试上传** 验证配置
7. 保存设置

:::tip
强烈建议在保存前点击「测试上传」，确认配置正确。测试会上传一张小图片并返回可访问的 URL。
:::

## 各服务配置

### 又拍云

| 参数 | 说明 |
|------|------|
| 服务名称 | 又拍云存储服务名 |
| 操作员 | 操作员账号 |
| 密码 | 操作员密码 |
| 上传路径 | 图片在存储中的路径前缀 |
| 自定义域名 | CDN 加速域名 |

### Cloudinary

| 参数 | 说明 |
|------|------|
| Cloud Name | Cloudinary 账户的 Cloud Name |
| API Key | API 密钥 |
| API Secret | API 密钥 |
| 上传路径 | 图片存储路径 |

### GitHub

| 参数 | 说明 |
|------|------|
| 仓库 | 格式为 `owner/repo` |
| 分支 | 存储分支（如 `main`） |
| Token | GitHub Personal Access Token |
| 上传路径 | 仓库中的存储路径 |
| 自定义域名 | CDN 域名（如 jsDelivr） |

### 自定义 S3 兼容

| 参数 | 说明 |
|------|------|
| Endpoint | S3 兼容的端点地址 |
| Access Key | 访问密钥 |
| Secret Key | 密钥 |
| Bucket | 存储桶名称 |
| 上传路径 | 存储路径前缀 |
| 自定义域名 | CDN 域名 |

## 工作流程

### 部署时自动上传

启用 CDN 后，每次部署时：

1. Gridea Pro 扫描项目中的所有图片资源
2. 对比已上传记录，识别新增或变更的图片
3. 自动上传到配置的 CDN 服务
4. 在渲染输出中将本地图片路径替换为 CDN URL

### 编辑器中上传

在文章编辑器中插入图片时，图片会先保存到本地项目。部署时由 CDN 模块统一处理上传。

## 注意事项

:::caution
CDN 凭据（API Key、密码等）安全存储在系统钥匙串中，不会出现在配置文件里。
:::

:::note
如果不启用 CDN，图片会随静态站点一起部署到目标平台。对于图片不多的小型博客，这通常已经足够。
:::

## 数据存储

CDN 配置保存在站点目录下的 `config/cdn_setting.json` 文件中。
