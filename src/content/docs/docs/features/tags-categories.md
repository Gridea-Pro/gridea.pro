---
title: 标签与分类
description: 使用标签和分类对博客内容进行多维度组织和管理。
---

标签（Tags）和分类（Categories）是 Gridea Pro 中两种主要的内容组织方式。它们可以单独使用，也可以配合使用来构建清晰的内容结构。

## 标签 vs 分类

| | 标签 | 分类 |
|---|------|------|
| 关系 | 扁平结构 | 可带层级 |
| 数量 | 一篇文章可有多个 | 一篇文章可有多个 |
| 用途 | 描述主题关键词 | 划分内容大类 |
| URL | `/tags/标签slug/` | `/categories/分类slug/` |
| 特色 | 支持自定义颜色 | 支持描述和 Slug |

## 标签管理

### 创建标签

1. 在左侧导航中点击 **标签** 进入管理页面
2. 点击 **新建标签** 按钮
3. 填写标签信息：
   - **名称** — 标签的显示名称
   - **Slug** — URL 友好的标识符，会自动从名称生成
   - **颜色** — 从预设的 11+ 种颜色中选择

:::tip
标签颜色会在前台主题中用于视觉区分，选择合适的颜色可以让标签云更加美观。
:::

### 编辑与删除

- 点击标签卡片可编辑名称、Slug 和颜色
- 删除标签时，已关联文章中的标签引用会自动移除
- 支持拖拽排序调整标签显示顺序

### 在文章中使用标签

在文章编辑器的设置面板中，可以从已有标签中选择或直接输入新标签名称。每篇文章可以关联多个标签。

## 分类管理

### 创建分类

1. 在左侧导航中点击 **分类** 进入管理页面
2. 点击 **新建分类** 按钮
3. 填写分类信息：
   - **名称** — 分类的显示名称
   - **Slug** — URL 路径标识，例如 `tech`、`life`
   - **描述** — 分类的简要描述（可选）

### 编辑与删除

- 点击分类可编辑其名称、Slug 和描述
- 删除分类时，文章中的分类关联会自动清除
- 分类的 Slug 变更后，对应的 URL 也会随之变化

### 在文章中使用分类

在文章编辑器的设置面板中选择分类。一篇文章可以同时属于多个分类。

## 主题中的展示

标签和分类的数据会在渲染时传递到主题模板中：

### 标签页面

主题可以渲染标签列表和标签归档页面。每个标签对象包含：

```
name    — 标签名称
slug    — URL 标识
color   — 标签颜色
count   — 关联文章数（由主题计算）
```

### 分类页面

每个分类对象包含：

```
id          — 唯一标识
name        — 分类名称
slug        — URL 标识
description — 分类描述
```

:::note
不同主题对标签和分类的展示方式可能不同。确保你使用的主题支持分类功能后再进行配置。
:::

## 最佳实践

- **标签宜精不宜多** — 控制标签数量，避免过于碎片化
- **分类保持稳定** — 分类代表博客的主要方向，不宜频繁变动
- **合理使用 Slug** — Slug 会出现在 URL 中，建议使用简短的英文
- **颜色有意义** — 给同类主题的标签分配相近颜色，增强视觉关联
