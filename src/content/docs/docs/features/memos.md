闪念笔记（Memos）是 Gridea Pro 内置的快速记录工具，适合捕捉碎片化的想法、灵感和素材。它独立于文章系统，更轻量、更即时。

## 核心特性

- **快速记录** — 打开即写，无需设置标题和元数据
- **标签系统** — 使用 `#标签名` 语法自动创建和关联标签
- **图片附件** — 每条闪念可添加图片
- **贡献热力图** — 可视化你的记录频率，类似 GitHub 贡献图
- **统计面板** — 查看总条数、各标签使用频次

## 创建闪念

1. 在左侧导航中点击 **闪念** 进入闪念页面
2. 在顶部输入框中输入内容
3. 使用 `#标签名` 在内容中添加标签，例如：

   ```
   今天发现一个很好的配色方案 #设计 #灵感
   ```

4. 点击图片按钮可添加图片附件
5. 点击发送按钮保存

:::tip
标签会自动从内容中提取，无需手动管理。输入 `#` 后直接跟标签文字即可。
:::

## 管理闪念

### 编辑与删除

- 点击闪念卡片上的编辑图标可修改内容
- 点击删除图标可删除该条闪念
- 编辑后标签会自动重新提取和更新

### 标签管理

在闪念页面的侧边栏中可以看到所有标签及其使用次数：

- **重命名标签** — 点击标签旁的编辑按钮，修改名称后会自动更新所有关联闪念
- **删除标签** — 删除标签时只移除标签关联，不会删除闪念内容
- **按标签筛选** — 点击标签名称可快速筛选该标签下的所有闪念

## 贡献热力图

闪念页面底部展示贡献热力图（Contribution Graph），以日历格式展示你每天的记录情况：

- 颜色深浅代表当天记录的闪念数量
- 帮助你保持写作习惯和记录节奏
- 数据基于每条闪念的创建时间自动生成

## 闪念与文章的关系

闪念和文章是两套独立的内容系统：

| | 闪念 | 文章 |
|---|------|------|
| 用途 | 碎片化记录、灵感收集 | 完整的博客内容 |
| 元数据 | 仅标签和时间 | 标题、分类、标签、特色图等 |
| 发布 | 渲染到闪念页面 | 渲染为独立文章页 |
| 编辑器 | 简单文本框 | Monaco 全功能编辑器 |

:::tip
借助 AI 助手的「闪念成文」功能，你可以将多条相关闪念整理为一篇完整的文章。详见 [AI 助手](/docs/features/ai-assistant)。
:::

## 数据存储

闪念数据保存在站点目录下的 `config/memos.json` 文件中，每条闪念包含：

- `content` — 文本内容
- `tags` — 自动提取的标签列表
- `images` — 图片附件路径
- `createdAt` — 创建时间
- `updatedAt` — 最后修改时间
